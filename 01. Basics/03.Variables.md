# Variables

- A program needs to store, read and modify data over time. Data lives in the memory.
- Variables are used to access data from the memory in a convinient way.

## Unitialized variable declaration

```go
var age int
fmt.Println(age)
```
- Unitialized variables take the default value based on the type.


## Initialized variable declaration

```go
var firstName = "saksham"
var lastName string = "Gupta"
```
- If we are initialzing variable, it's optional to specift type as it will infer from the value assigned.

## Multiple variables declaration

```go
func main() {
	var a, b int = 10, 20
	fmt.Println(a, b)
}
```


## Shorthand

- No need to write `var` keyword and define the type as it will be infered from the value.
- This syntax is only valid inside functions. Package level variables are still declared using `var` keyword.

```go
count := 10
```

## Default values

- If we leave variables unitialized, they take a default value based on the data type.

| Go Data Type     | Default Value                         |
| ---------------- | ------------------------------------- |
| `bool`           | `false`                               |
| `string`         | `""` (empty string)                   |
| `int`            | `0`                                   |                           |
| `array`          | All elements set to their zero values |
| `struct`         | All fields set to their zero values   |
| `pointer`        | `nil`                                 |
| `slice`          | `nil`                                 |
| `map`            | `nil`                                 |
| `channel`        | `nil`                                 |
| `function`       | `nil`                                 |
| `interface`      | `nil`                                 |


## Naming conventions

- For types, interfaces, enums, structs we use PascalCase.
- For constants we use UPPERCASE.
- For naming files, we can use snake_case.
- For variables, we can use snake_case or camelCase.
- Package names should be lowercase.
