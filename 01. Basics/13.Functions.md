# Functions

- To organise our long code into modular and reusable blocks that does exactly one thing based on single responsibility principle.
- It improved code readability and maintainability.

## Syntax

- func reserved keyword.
- name of the function. If it's a `public` function, it should be capitalCased. `Private` ones are lowercased.
- List of parameters with their types (p int, q string).
- Return type/s. Function can return multiple values.

```go
func greetUser(name string) string {
  fmt.Println("WELCOME", name)
}
```

## Anonymous functions

- Functions that do not have a name.
- We call these functions by assigning them to a variable first and then call them using variable.

```go
func main() {
	var result = func() {
		fmt.Println("Hello world")
	}
	result()
}
```

## IIFE

- A function that invokes itself while declaration.

```go
func main() {
	func() {
		fmt.Println("Hello world")
	}()
}
```

## Functions as types: First class citizens

- Just like any other basic type (string, boolean, int) in Go, we can do the following with functions
- Assign them to a variable.
- Pass them as arguments.
- Return them from a function.
- Store them in data structures like arrays, maps etc

### Assigning functions to a variable just like other types(string)

```go
func main() {

	adder := add
	adder(1, 2)

}

func add(a int, b int) int {
	return a + b
}
```

### Passing functions as arguments

```go
func main() {
	result := applyOperation(1, 2, add)
	fmt.Println(result)

}

func applyOperation(x int, y int, cb func(int, int) int) int {
	return cb(x, y)
}

func add(a int, b int) int {
	return a + b
}
```

### Returning functiion from function body

```go
func main() {
	multiplyByTwo := multiplier(2)
	fmt.Println(multiplyByTwo(6))

}

func multiplier(x int) func(int) int {
	return func(y int) int {
		return x * y
	}
}
```
