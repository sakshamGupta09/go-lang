# Range

- It's a convinient function that lets us iterate over collections like arrays, slices, maps, channels, strings.
- It provides a uniform way of iterating over collections for which otherwise we would need different syntax for different collections.
- Range creates a copy of each element. So, modifying this copy will not impact the original element.

## Syntax

- `for i,v := range collection`

```go
func main() {
	var list = []int{1, 2, 3, 4, 5}

	for i, v := range list {
		v = v * 2
		fmt.Println(i, v)
	}

	fmt.Println(list) // [1 2 3 4 5]

}
```


## Iterating over strings

- The value returned is the uniocode value of character.

```go
func main() {
	message := "Hello World"

	for i, v := range message {

		fmt.Println(i, v)

		/*
		   0 72
		   1 101
		   2 108
		   3 108
		   4 111
		   5 32
		   6 87
		   7 111
		   8 114
		   9 108
		   10 100
		*/
	}
}
```


## Blank identifier '_'

- Whenever a function is returning multiple values like range but we do not want to use a particular value, we can use _ instead.
- We do not have to reserve memory for that variable.
- Linter error for unused variable will go away.

```go
func main() {
	var list [3]int8 = [3]int8{1, 2, 3}

	for _, value := range list {
		fmt.Println(value)
	}
}
```

```go
func main() {
	_, j := getNums()
	fmt.Println(j)
}

func getNums() (int, int) {
	return 1, 2
}
```
