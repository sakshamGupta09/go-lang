# Arrays

- To store multiple values.
- Fixed size and each element will have same type. While declaration, we need to specify both of these properties. So, beforehand we must know about the size and the type.
- Index starts with 0 and last index is n-1.

## Declaration

```go
func main() {
	var fruits [5]string
}
```
- If we do not initialize the array, the elements will take a default value.

## Initialization

```go
func main() {
	var nums [5]int8 = [5]int8{1, 2, 3, 4, 5}

	fmt.Println(nums)
}

func main() {
	nums := [5]int8{1, 2, 3, 4, 5}

	fmt.Println(nums)
}
```

## Arrays are passed as value

- When we assign an array to a new variable or pass them as arguments, A new array is created by copying the value from original array.
- Both original and new are pointing to different places in memory.

```go
func main() {
	var originalList [3]int8 = [3]int8{1, 2, 3}
	var copy = originalList

	copy[0] = 0

	fmt.Println("ORIGINAL ARRAY", originalList) // ORIGINAL ARRAY [1 2 3]
	fmt.Println("COPY ARRAY", copy) // COPY ARRAY [0 2 3]
}
```

## Iterating over arrays

### Traditional for loop

- Since index starts at 0 and ends at n-1, so will our loop.
  
```go
func main() {
	var list [3]int8 = [3]int8{1, 2, 3}

	for i := 0; i < len(list); i++ {
		fmt.Println(i, list[i])
	}
}
```

### Range

```go
func main() {
	var list [3]int8 = [3]int8{1, 2, 3}

	for index, value := range list {
		fmt.Println(index, value)
	}
}
```

## Blank identifier '_'

- Whenever a function is returning multiple values like range but we do not want to use a particular value, we can use _ instead.
- We do not have to reserve memory for that variable.
- Linter error for unused value will go away.

```go
func main() {
	var list [3]int8 = [3]int8{1, 2, 3}

	for _, value := range list {
		fmt.Println(value)
	}
}
```
