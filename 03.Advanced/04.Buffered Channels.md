# Buffered Channels

```go
ch := make(chan int, 2)
```

- Has storage to store some number of messages. So the commmunication model is now changed. Earlier to talk, sender and receiver both were needed. Now sender can drop messages into the buffer and move on with their other activities. Receiver will read messages from buffer and keeo doing their activities. No blocking for each other is needed now.
- Can be viusalized as a phobe call communication vs text sms.
- What if buffer gets full? In that case to talk, again sender and receiver needs to come together and wait for each other.
- When buffer gets empty? Receiver will have to wait for sender to come and communicate.
- When we read from the channel, first value is removed and slot becomes available.


| Feature         | Unbuffered                    | Buffered               |
| --------------- | ----------------------------- | ---------------------- |
| Storage         | ❌ None                        | ✅ Yes                 |
| Send blocks?    | Always (until receiver ready) | Only when buffer full  |
| Receive blocks? | Always (until sender ready)   | Only when buffer empty |
| Use case        | Synchronization               | Data passing / queues  |


## Non blocking 

```go
func main() {
	ch := make(chan int, 2)

	ch <- 10
	fmt.Println("Value sent")
}
```
- Since buffer is not full, send does not wait for anything and doesn't blocks execution.
- With unbuffered channels, we were not able to do this. Sender blocks and waits for the receiver.

```go
func main() {
	ch := make(chan int, 2)

	ch <- 1
	ch <- 2

	fmt.Println("RECEIVER 1", <-ch)
	fmt.Println("RECEIVER 2", <-ch)

	ch <- 3
	fmt.Println("RECEIVER 3", <-ch)
}
```

```go
func main() {
	ch := make(chan int, 2)

	ch <- 1
	ch <- 2
	ch <- 3

	fmt.Println("Value sent")
}
```
- Now the sender will block and wait for the receiver because we have exceeded capacity and cannot hold the values anymore in the buffer.

```go
func main() {
	ch := make(chan int, 2)

	ch <- 1
	ch <- 2

	go func() {
		time.Sleep(2 * time.Second)
		fmt.Println("RECEIVED", <-ch)
	}()

	ch <- 3
	fmt.Println("Value sent")
}
```
- While sending the third value, sender will block as buffer is full.
- After 2 seconds, we get a receiver. So value one moves out of the buffer and 3 is written to it.


```go
func main() {
	ch := make(chan int, 2)
	<-ch
}
```
- Receiver blocks if the buffer is empty as there is nothing to read.

```go
func main() {
	ch := make(chan int, 2)

	go func() {
		time.Sleep(2 * time.Second)
		ch <- 1
	}()

	fmt.Println(<-ch)
}
```
