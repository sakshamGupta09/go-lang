# HTTP Versions


## HTTP

- Consider a client server architecture. Client sends a request to the server and server responds.
- This communication happens using a protocol called HTTP.
- A protocol is simply a set of rules for communication.
- First a TCP connection is established between client and server and then request response follows.


## HTTP/1.1

- I want to cummunicate with you several times. I have a lot of requests to make.
- Okay, for every request create a new connection with me.
- Lets say, we ask our browser to render a webpage. The webpage will need resources like HTML, CSS, JS and images to render. So, browser which is a client will request server to download files.
- For every file, we need to create a new connection and then send request to server.

```go
Open connection
Request
Close connection
```

### keep-alive connections.

- To improve performance, for every file, it can reuse the same connection.
- In HTTP/1.1, requests on the same connection are processed in order.

```go
Open connection
Request HTML
Request CSS
Request JS
Request Image
Close connection
```

### Limitations

- Requests still block each other on a connection.
- If one request is slow, others may have to wait.



## HTTP/2

- Newer version designed to solve the performance problems of HTTP/1.1
- It supports multiplexing. A single connection can carry multiple requests simultaneously. Requests are not processed in order.


## HTTPS

- HTTPS is for the secure communication between client and the server.
- HTTPS = HTTP + TLS encryption.
- Client and server perform a TLS handshake first and then start encrypted communication.
