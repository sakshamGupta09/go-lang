# Code organisation and structure

- For any large and complex project, there needs to be some structure and organisation of code into logical units so it is easier to understand and manage it.

## Packages

- A package is the smallest unit of code organisation. It groups extremely related go files which can contain functions, variables and types.
- Every GO file belongs to a pakage.
- Each package can be imported and used in other package. Packages promote reusability.
- Example: Consider the built in packages offered by the standard library.


## Modules


- A module is a collection of related GO packages that tracks it's dependencies it needs to run correctly.
- Unit for versioning and depnedency management.

## go mod init name

- It initialises a new GO module and creates a `go.mod` file.
- This file contains name of the module, GO version it uses and a list of external dependencies with their versions.

## Some useful commands

| Command         | Action                                                                        |
| --------------- | ----------------------------------------------------------------------------- |                                           
| go mod tidy     | Removes the unused dependencies from the mod file and adds the missing ones.  |
| go get deps     | Adds or updates external dependency.                                          |
