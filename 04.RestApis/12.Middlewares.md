# Middleware

- We want to process our request in stages before it can reach the handler.
- Each middleware receives a request, does some action on that request and forwards it to the next middleware or handler in line.
- Once, next is called we cannot tamper the request or write object anymore as its already passed to the next stage.
- Logging, Validation, Authentication and Authorization, Error Handling, Rate limiting.

```go
func SecurityHeaders(next http.Handler) http.Handler {

	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		next.ServeHTTP(w, r)
	})
}
```


## Security Headers

- Create a custom middleware that receives a request and sets security headers in the response.

```go
func SecurityHeaders(next http.Handler) http.Handler {

	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		w.Header().Set("Header name", "Header value")
		next.ServeHTTP(w, r)
	})
}
```

## CORS

- A security feature implemented at browser.
- If a webpage at domain A is trying to reach to a server at domain B, browser will ask the server if the serve is open to receiving request from this domain.(Usually happens with the response header Access-Control-Allow-Origin)
- If server says YES, it works. If server denies, browser will throw an error and not let us read the information.

```go
func Cors(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		origin := r.Header.Get("Origin")

		if !isOriginAllowed(origin) {
			http.Error(w, "Forbidden", http.StatusForbidden)
			return
		}
		w.Header().Set("Access-Control-Allow-Origin", origin)
		next.ServeHTTP(w, r)
	})
}
```

## ResponseTime

- How much time we too to process the request. Time taken from the moment request reached us and the moment we send the response.

```go
func ResponseTime(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		fmt.Println("Received request in middleware")
		start := time.Now()

		rw := &CustomResponseWriter{
			statusCode:     http.StatusOK,
			ResponseWriter: w,
		}

		next.ServeHTTP(rw, r)
		duration := time.Since(start)
		fmt.Printf("METHOD: %s, URL: %s, statusCode: %d, Duration:%v", r.Method, r.URL, rw.statusCode, duration)
		fmt.Println("Response sent from middleware")
	})
}

type CustomResponseWriter struct {
	statusCode int
	http.ResponseWriter
}

func (rw *CustomResponseWriter) WriteHeader(code int) {
	rw.statusCode = code
	rw.ResponseWriter.WriteHeader(code)
}

```

- The inbuilt `ResponseType` does not expose all the information we need, so every stage can just add some information and pass it forward.


## Compression

- Our server is listening to incoming requests and will send them response.
- Compression reduces the size of the response body that can help us improve performance.
- Not all clients can understand compressed data. So client sends us the encoding it understands using request header (Accept-Encoding).
- Based on that, server will respond back by sending header (Content-Encoding) which tells client data is encoded using which technique.
- Compressing data takes CPU, So if server is already under immense load, it might not be worth it.

```go
func Compression(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		acceptedEncoding := r.Header.Get("Accept-Encoding")
		fmt.Println(acceptedEncoding)
		if !strings.Contains(acceptedEncoding, "gzip") {
			next.ServeHTTP(w, r)
			return
		}
		w.Header().Set("Content-Encoding", "gzip")
		gzWriter := gzip.NewWriter(w)
		defer gzWriter.Close()

		rw := &GzipResponseWriter{ResponseWriter: w, Writer: gzWriter}

		next.ServeHTTP(rw, r)
	})
}

type GzipResponseWriter struct {
	http.ResponseWriter
	Writer *gzip.Writer
}

func (g *GzipResponseWriter) Write(b []byte) (int, error) {
	return g.Writer.Write(b)
}
```
