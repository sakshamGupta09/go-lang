# Add certificates to our server

- We have spawned our server and asked users to come with their requests.
- But while clients are trying to reach our servers, they are getting a warning that our server is not secure.
- So, we need to add certificates to our server.

## SSL/TLS certificates

### Encryption — Keep data private

- Without a certificate, anyone between you and the server can read the traffic.
- A TLS certificate allows the browser and server to establish an encrypted connection so that intercepted data looks like random gibberish.

###  Authentication — Prove the server's identity

- To prove that you're talking to the correct server and not an impersonator.
- Imagine you want to connect to: `https://mybank.com`. How do you know you're really talking to your bank and not an attacker pretending to be the bank?
- A TLS certificate contains the server's identity and is digitally signed by a trusted Certificate Authority (CA).
