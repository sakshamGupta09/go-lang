# Sorting

- `Sort` package in go helps in ordering collections. It can be collection of numbers, strings, floats or even custom structs.
- Sorting happens in place. Original collection is modified.


## Sort integers

```go
func main() {
	nums := []int{1, 5, 3, 2, 4}

	sort.Ints(nums)
	fmt.Println(nums)
}
```

## Sort Strings

```go
func main() {
	fruits := []string{"grapes", "orange", "apple", "banana"}

	sort.Strings(fruits)
	fmt.Println(fruits)
}
```

## Sort Floats

```go
floats := []float64{3.2, 1.5, 2.8}
sort.Float64s(floats)
```

## Check if already sorted

```go
sort.IntsAreSorted(nums) // true or false
```

## Custom sort

- Define custom rules for sorting the data.


## sort.Slice

- You provide a comparison function.
- Function takes two indices and returns true if element at i should appear before element at index j.

```go
type Person struct {
	Name string
	Age  int
}

people := []Person{
	{"Alice", 25},
	{"Bob", 20},
	{"Charlie", 30},
}

// sort by Age
sort.Slice(people, func(i, j int) bool {
	return people[i].Age < people[j].Age
})
```

Multi field sorting

```go
sort.Slice(people, func(i, j int) bool {
	if people[i].Age == people[j].Age {
		return people[i].Name < people[j].Name
	}
	return people[i].Age < people[j].Age
})
```


## sort.Interface

- You implement 3 methods:

```go
type ByAge []Person

func (a ByAge) Len() int           { return len(a) }
func (a ByAge) Less(i, j int) bool { return a[i].Age < a[j].Age }
func (a ByAge) Swap(i, j int)      { a[i], a[j] = a[j], a[i] }
```

```go
sort.Sort(ByAge(people))
```

- We need to sort people but it does not obey the interface. So, we typecast it.


## Stable vs Unstable Sort

- Order for equal items should remain as per the original array or not.
- `sort.Slice` is not stable sort.
- `sort.SliceStable` is stable .
