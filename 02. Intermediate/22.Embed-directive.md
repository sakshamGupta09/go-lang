# Embed directive

- Let's say our program depends upon some static files like `email-templates`, `sql-migrations`, `HTML/CSS`, `assets` etc. Without embedding, these files must exist on the filesystem at runtime.
- While deployment, we need to ensure that the static folders also exist. If we run binary from a different folder, it will crash.
- Packaging complexity, instead of single executable we now need to ship binary along with all the other folders.
- embed allows you to compile files into the binary itself.

## Syntax

```go
import (
	_ "embed"
	"fmt"
)

//go:embed file.txt
var content string
```
- This tells Go that this file needs to be part of the executable produced.
- We are not directly using methods or types from `embed` package but indirectly so the blank import.


## Embedding folders

```go
import (
	"embed"
	"fmt"
)

//go:embed docs
var folder embed.FS

func main() {
	content, err := folder.ReadFile("docs/hello.txt")

	if err != nil {
		fmt.Println("ERROR::", err)
		return
	}
	fmt.Println(string(content))
}

```

## Navigating the folder

```go
func main() {

	fs.WalkDir(folder, "docs", func(path string, d fs.DirEntry, err error) error {
		if err != nil {
			return err
		}
		fmt.Println(path)
		return nil
	})
}
```
