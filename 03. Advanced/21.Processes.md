
# Analogy

- Think of all programs on your OS as running restaurants.
- Goroutines are workers inside the same kitchen of one resturtant.
- Processes are completely separate restaurants.

## Goroutines

- When we run our program, a process is created and within that single process we have created multiple goroutines(workers) that are working concurrently.

## Processes

- A process is an entirely separate running program.
- Your GO app runs and it starts anothe rprogram. This is called process spawning.
- Process spawning means Ask the operating system to start another independent program.
- Your Go program asked the OS: “Please create another process that runs echo Hello.”

```go
package main

import (
	"fmt"
	"os/exec"
)

func main() {
	cmd := exec.Command("echo", "Hello")

	output, _ := cmd.Output()

	fmt.Println(string(output))
}
```

## Why use another process instead of goroutines

Because sometimes you want:

- isolation
- safety
- independent resources
- external tools
