
# Analogy

- Think of a Go program as a running restaurant.
- Goroutines are workers inside the same kitchen.
- Processes are completely separate restaurants.

## Goroutines

- Goroutines (Concurrency inside same process). When we create a goroutine, all goroutines are running inside a same process.
- All goroutines share same memory and CPU resources because of which one bug can affect the other goroutine. A crash can crash the whole process.

## Processes

- A process is an entirely separate running program.
- Your GO app runs and it starts anothe rprogram. This is called process spawning.
- Process spawning means Ask the operating system to start another independent program.
- Your Go program asked the OS: “Please create another process that runs echo Hello.”

```go
package main

import (
	"fmt"
	"os/exec"
)

func main() {
	cmd := exec.Command("echo", "Hello")

	output, _ := cmd.Output()

	fmt.Println(string(output))
}
```
