# Channels

- Once you start a goroutine using go, It runs separately from the main flow of your program and Go does not guarantee:when it will run, in what order it will run and when it will finish.
- So, we need a mechanism to synchronize and communicate between goroutines.
- Suppose we have producer, consumer pattern. One goroutine produces data and the other consumes and processes it.

## Creating a Channel

```go
ch := make(chan int)
```

## Sending Data to a Channel

- Pipe is towards a channel. So, data is going inside a channel.

```go
ch <- 10
```

## Receiving Data from a Channel

```go
value := <-ch
```
