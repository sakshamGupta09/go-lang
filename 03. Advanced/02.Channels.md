# Channels

- Once you start a goroutine using go, It runs separately from the main flow of your program and Go does not guarantee:when it will run, in what order it will run and when it will finish.
- So, we need a mechanism to synchronize and communicate between goroutines.

## Creating a Channel

```go
ch := make(chan int)
```

## Sending Data to a Channel

- Pipe is towards a channel. So, data is going inside a channel.

```go
ch <- 10
```

## Receiving Data from a Channel

```go
value := <-ch
```

## Example

```go
package main

import "fmt"

func sendData(ch chan int) {
	ch <- 42
}

func main() {

	ch := make(chan int)

	go sendData(ch)

	value := <-ch

	fmt.Println(value)
}
```
- Here we have two goroutines `main` and `sendData`.
- `main` goroutine will be blocked and wait till channel sends some data because it needs to process that data so it has to wait.
- Channels are blocking by default. Send blocks until receiver is ready. Receive blocks until sender sends.
- Channels allow communication between goroutines only. It works because `main` is also a goroutine.


```go
func sendData(ch chan int) {
	fmt.Println("SENDING DATA ")
	time.Sleep(time.Second * 2)
	ch <- 10
}

func main() {
	ch := make(chan int)

	go sendData(ch)

	fmt.Println("Waiting for DATA ")
	time.Sleep(5 * time.Second)
	value := <-ch

	fmt.Println(value)
}
```

## Receiving multiple values

- If sender sends multiple data, we need to reexecute receiver command. Otherwise it will just listen to the first emission.
- Channels do not broadcast values to all receivers. One send , one receive.
- For every value send, we need to have equal number of receivers.

```go
func sendData(ch chan int) {
	ch <- 10
	fmt.Println("send data GOROUTINE")
}

func receiveData(ch chan int) {
	value := <-ch

	fmt.Println("RECEIVER GOROUTINE", value)
}

func main() {
	fmt.Println("START PROGRAM")

	ch := make(chan int)

	go sendData(ch)
	go receiveData(ch)

	receiver := <-ch
	fmt.Println(receiver, "MAIN GOROUTINE")

	time.Sleep(2 * time.Second)
}
```
