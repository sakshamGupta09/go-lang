# Channels

- To enable communication between goroutines.
- Suppose one goroutine produces data and the othe one consumes and then processes that data. How do we make sure data is available for goroutine2.
- A channel is a pipe for communication between goroutines. Using this pipe, multiple goroutines can send and receive data.
- Channels are typed, meaning they carry a specific type of data.


## Creating a Channel

```go
ch := make(chan int)
```

## Sending Data to a Channel

- Pipe is towards a channel. So, data is going inside a channel.

```go
ch <- 10
```

## Receiving Data from a Channel

```go
value := <-ch
```

## Example

```go
package main

import "fmt"

func sendData(ch chan int) {
	ch <- 42
}

func main() {

	ch := make(chan int)

	go sendData(ch)

	value := <-ch

	fmt.Println(value)
}
```
- Here we have two goroutines `main` and `sendData`.
- `main` goroutine will be blocked and wait till channel sends some data because it needs to process that data so it has to wait.
- Channels are blocking by default. Send blocks until receiver is ready. Receive blocks until sender sends.

```go
func sendData(ch chan int) {
	fmt.Println("SENDING DATA ")
	time.Sleep(time.Second * 2)
	ch <- 10
}

func main() {
	ch := make(chan int)

	go sendData(ch)

	fmt.Println("Waiting for DATA ")
	time.Sleep(5 * time.Second)
	value := <-ch

	fmt.Println(value)
}
```
