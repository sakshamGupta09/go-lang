# sync.Cond

- Goroutines need to wait for some condition to become true.
- A sync.Cond has 3 important methods and it always works with a mutex.

| Method        | Meaning                     |
| ------------- | --------------------------- |
| `Wait()`      | Sleep until notified        |
| `Signal()`    | Wake one waiting goroutine  |
| `Broadcast()` | Wake all waiting goroutines |


## Wait

Wait methods does three things -

- Unlocks the mutex. I am going to sleep now, but I will release teh lock before going. Other goroutines can now modify data.
- Puts goroutine to sleep. No CPU wasted.
- When signaled, wakes up, re-locks mutex automatically and continues execution.


## Why use for loop around Wait?

Always do

```go
for !condition {
    cond.Wait()
}
```
NOT

```go
if !condition {
    cond.Wait()
}
```

- Because waking up does NOT guarantee condition became true.
- So after waking up, check again and if condition still false → sleep again

## Basic Example

```go
package main

import (
	"fmt"
	"sync"
	"time"
)

func main() {
	var mu sync.Mutex
	cond := sync.NewCond(&mu)

	ready := false

	// Waiting goroutine
	go func() {
		mu.Lock()

		for !ready {
			fmt.Println("Waiting...")
			cond.Wait()
		}

		fmt.Println("Received signal!")
		mu.Unlock()
	}()

	time.Sleep(2 * time.Second)

	// Producer goroutine
	mu.Lock()
	ready = true
	fmt.Println("Sending signal...")
	cond.Signal()
	mu.Unlock()

	time.Sleep(time.Second)
}
```

## Step-by-step intuition

- Goroutine locks the mutex and reads shared state safely.
- Called the `Wait` method which puts the goroutine to sleep, unlocks the mutex and waits for the signal to come.
