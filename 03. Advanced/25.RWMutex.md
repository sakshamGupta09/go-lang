# RWMutex


## Problem with sync.Mutex

- We define a critical section. Only one goroutine can access this critical section at a time.
- Others will have to wait unless the lock is released. Once a mutex is locked no other goroutine can lock it unless it opens first.


## RWMutex solves this

- Multiple readers can read together instead of waiting one by one.
- Only one writer allowed
- Writer blocks everyone else

## Mental Model

### Reader

- Multiple readers can read data together because nobody is modifyng that data.
- In normal mutex, lock can be acquired by only one goroutine.
- For RWMutex, multiple goroutines can acquire read lock on mutex.

### Writer

- When a writer is modifying data, all readers and other writers are blocked.
