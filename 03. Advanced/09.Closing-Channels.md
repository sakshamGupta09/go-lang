# Closing channels

- We close channels to indicate completion. That no more data will be send now.
- So, goroutines thta are waiting for this data can stop waiting.
- We cannot send values after a channel is closed. But we can receive values from a closed channel if it's a buffered channel.
- Only goroutine that is sending data should close a channel.
- Channels should be closed once. If we close a closed channel, panic happens.

```go
func main() {
	ch := make(chan int)

	go func() {
		for i := range 5 {
			ch <- i
		}
		close(ch)
	}()

	for value := range ch {
		fmt.Println("INSIDE LOOP RANGE")
		fmt.Println(value)
	}
}
```

```go
func main() {
	ch := make(chan int)

	close(ch)

	val, ok := <-ch
	if !ok {
		fmt.Println("Channel is already closed")
		return
	}

	fmt.Println("Receiving value form channel", val)
}
```
