# Directions

- Channels are bidirectional. We can send values to a channel as well as receive values from it.
- When we pass this channel variable to a producer or a consumer, it's purpose is not quite clear. Also there is no compile time type safety.
- For sake of type safety and readability, I want to classify my channels as sender and receiver.

```go
chan<- int   // send-only channel
<-chan int   // receive-only channel
```

## Prevents bugs at compile time

- Without direction, if a function receives a channel as a parameter, It's not clear if I should use that channel to send data or receive.
- With directions, we get compile time errors. This ensures that sender does not accidently receives and receiver does not accidently sends.

```go
func process(ch chan int) {
	ch <- 10      // maybe unintended
	value := <-ch // maybe unintended
}
```

```go
func sender(ch chan<- int) {
	ch <- 10   // ✅ allowed
	// <-ch ❌ compile error
}

func receiver(ch <-chan int) {
	value := <-ch // ✅ allowed
	// ch <- 10 ❌ compile error
}
```

## Makes code self-documenting

```go
func fetchData(ch chan<- int)
```
- This function only sends data.


## Direction is only a restriction, not a new channel

```go
ch := make(chan int)

var sendOnly chan<- int = ch
var recvOnly <-chan int = ch
```
All refer to the same underlying channel


## You can convert bidirectional → directional

```go
func send(ch chan<- int) {}
func receive(ch <-chan int) {}

func main() {
	ch := make(chan int)

	send(ch)    // allowed
	receive(ch) // allowed
}
```

## Works perfectly with range

- range only needs receive capability.

```go
func consumer(ch <-chan int) {
	for val := range ch {
		fmt.Println(val)
	}
}
```

## Closing channels

- Only the sender should close the channel

```go
func producer(ch chan<- int) {
	ch <- 1
	close(ch) // ✅ correct
}
```

## Examples

```go
func producer(ch chan<- int) {
	for i := 1; i <= 3; i++ {
		ch <- i
	}
	close(ch)
}

func consumer(ch <-chan int) {
	for val := range ch {
		fmt.Println("Received:", val)
	}
}

func main() {
	ch := make(chan int)

	go producer(ch)
	consumer(ch)
}
```
