# Testing

- Testing is the process to verify that our code is working as expected. It does what we expect it to do.
- Ensure the firther changes to our codebase do not break any existing implementations.
- Documentation on how the code should function.

## Prerequisites

- File suffix must be `_test.go`.
- Command to run the test file is `go test filename`.


## Wrtite a simple test case

- First we need to write a testing function that will test a real function's implementation.
- A testing function's name must start with prefix `Test` and it must take a parameter `(*testing.T)`.
- We call the real function and get the returned value. We compare this value with the expected value and base don that fail or pass teh test.

```go
func TestAdd(t *testing.T) {
	result := Add(1, 2)
	expected := 5

	if result != expected {
		t.Errorf("Test failed for Add(%d, %d). Expected value %d return value %d", 1, 2, expected, result)
	}
}
```

## Table Driven Tests

- I want to test my function by running multiple test cases with varying inputs.
- We will create a slice of test inputs and expected values.
- Then in a loop we call the real function with input test parameters and compare the result with the expected value.

```go
func TestAdd(t *testing.T) {
	tests := []AddTestData{
		{1, 2, 3},
		{0, -1, -1},
		{-2, -2, -4},
	}
	for _, test := range tests {
		result := Add(test.a, test.b)
		if result != test.expected {
			t.Errorf("Add(%d, %d) = %d, expected %d", test.a, test.b, result, test.expected)
		}
	}
}
```
