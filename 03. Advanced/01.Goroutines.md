# Goroutines

- I want to perform 3 tasks. One way is to complete Task 1 first, then Task 2 and then Task 3. Here each task waits for the other one to complete.
- The better approach is to perform all three tasks concurrently(at the same time) if they are independent
- A Goroutine is a lightweight thread managed by the Go runtime.

## Example

```go
package main

import (
	"fmt"
	"time"
)

func printNumbers() {
	for i := 1; i <= 5; i++ {
		fmt.Println(i)
		time.Sleep(time.Millisecond * 500)
	}
}

func main() {
	go printNumbers()

	fmt.Println("Main function")

	time.Sleep(time.Second * 3)
}
```

- `go printNumbers()` starts a goroutine
- `main()` continues running immediately
- Both run concurrently


## Main May Exit Early

- If `main()` finishes, all goroutines stop immediately.

```go
func main() {
	go sayHello()
}

func sayHello() {
	time.Sleep(time.Second * 1)
	fmt.Println("Hello from goroutine")
}

```
- It will not print anything. Because `sayHello` create a new goroutine( a lightweight thread) and main and `sayHello` executes concurrently.
- But the `main` function finished execution and program stopped.


## Concurrency vs Parallelism

- You handle multiple tasks at the same time, but not necessarily executing them simultaneously. You switch between tasks quickly. Only one task runs at a time, but progress happens for all.
- Parallelism means Multiple tasks run at the exact same time.


## What Happens When You Start Goroutines

```go
go task1()
go task2()
go task3()
```

### Case 1 — Single CPU

```go
CPU
 ↓
task1
task2
task3
task1
task2
```
- This is concurrency (not parallelism). Only one goroutine runs at a time.


### Case 2 — Multiple CPUs

```go
CPU1 → task1
CPU2 → task2
CPU3 → task3
```
