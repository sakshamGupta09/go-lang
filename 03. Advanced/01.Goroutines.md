# The Problem: Doing multiple tasks

- We need to perform multiple tasks in our Go program that are long running.
- One way is **sequential execution**:
  - Task 1 → Task 2 → Task 3
  - Each task waits for the previous one.
- This can make the program slow and inefficient.
- So, we need a better way to handle multiple tasks.


## Concurrency (Structure)

- Concurrency means **managing multiple tasks at the same time**.
- Tasks are **in progress together**, but not necessarily running at the exact same moment.
- At any instant, only **one task may execute** (on a single core).
- The system switches between tasks quickly → gives illusion of parallel work.

✅ Key idea: *Dealing with multiple tasks together*


## Parallelism (Execution)

- Parallelism means **actually running tasks at the same time**.
- Requires **multiple CPU cores**.

✅ Key idea: Doing multiple tasks at the same time

```go
Core 1 → Task A
Core 2 → Task B
Core 3 → Task C
```


## What are Goroutines?

- Goroutines are lightweight threads managed by Go runtime.
- They allow functions to run concurrently.
- The go keyword starts a new goroutine.

## main is also a goroutine

- The main function runs in a goroutine called the main goroutine.
- If main exits → all goroutines stop immediately.

## Goroutines are NOT automatically synchronized

- Goroutines run independently.
- No guarantee of order or completion.
