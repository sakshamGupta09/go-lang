# Sequential Execution

- Let's say we need to perform 3 tasks.
- Normally our program executes one task at a time. It will complete task1 first then move to Task 2 and then Task 3.
- This makes sense if tasks are dependent on previous task's output but if they are independent then it creates performance issues.

## Concurrency

- Lets say we have 3 tasks to perform.
- Perform a task for a while then switch to the other one and so on so that all of them are progressing together.


## How to do concurrency in GO

- Goroutines
- Goroutine is a function that runs independently.

## Main exit

- If our main function exits, all goroutines are terminated.
