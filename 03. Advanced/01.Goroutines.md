# Goroutines

- I want to perform 3 tasks. One way is to complete Task 1 first, then Task 2 and then Task 3. Here each task waits for the other one to complete.
- The better approach is to perform all three tasks concurrently(at the same time) if they are independent

## Example

```go
package main

import (
	"fmt"
	"time"
)

func printNumbers() {
	for i := 1; i <= 5; i++ {
		fmt.Println(i)
		time.Sleep(time.Millisecond * 500)
	}
}

func main() {
	go printNumbers()

	fmt.Println("Main function")

	time.Sleep(time.Second * 3)
}
```

- `go printNumbers()` starts a goroutine
- `main()` continues running immediately
- Both run concurrently


## Goroutines are Cheap

- A Goroutine is a lightweight thread managed by the Go runtime.
- Traditional threads are heavy.
- Because goroutines are lightweight, Go programs can run thousands or even millions of them.

| Feature       | Thread    | Goroutine  |
| ------------- | --------- | ---------- |
| Memory        | ~1 MB     | ~2 KB      |
| Creation cost | Expensive | Very cheap |
| Management    | OS        | Go runtime |


## Main May Exit Early

- If `main()` finishes, all goroutines stop immediately.
