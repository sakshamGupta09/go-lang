# Sequential Execution

- We need to complete 3 tasks.
- In sequential execution, It will complete task1 first then move to Task 2 and then Task 3.
- This makes sense if tasks are dependent on previous task's output but if they are independent then it creates performance issues.

## Concurrency

- Lets say we have 3 tasks to complete which are independent from each other.
- Handling multiple tasks but doing one thing at a time.
- Switch between taks making progress on all of them.
- If a task is taking time or is waiting for external system, instead of keep waiting switch to some other task.


## How to do concurrency in GO

- Goroutines
- Goroutine is a function that runs independently.
- While calling a function we prefix it with `go` keyword that marks it as a goroutine. Goroutine is just a function that will be handled concurrently.

## Main exit

- If our main function exits, program completes and all goroutines are terminated as well.
- But they might still remain in the memory leading to memory leaks or goroutine leaks.


## Example

```go
func main() {
	printNumbers()
	printLetters()
}

func printNumbers() {
	for i := range 5 {
		fmt.Println(i)
		time.Sleep(time.Millisecond * 100)
	}
}

func printLetters() {
	for _, char := range "abcde" {
		fmt.Println(string(char))
		time.Sleep(time.Millisecond * 200)
	}
}
```
- By the time we are waiting for some milliseconds to pass, our CPU is idle.

```go
func main() {
	go printNumbers()
	go printLetters()

	time.Sleep(time.Second * 3)

	// a01b23c4de
}
```

## Return values from gorotines

- Normal functions can return some values. But goroutines cannot return values.

```go
func main() {
	var value int

	go func() {
		value = doWork()
		fmt.Println(value)
	}()
	time.Sleep(time.Second * 3)
}

func doWork() int {
	time.Sleep(time.Second * 2)
	return 10
}
```
