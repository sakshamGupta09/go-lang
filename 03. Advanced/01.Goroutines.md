# The Problem: Doing multiple tasks

- We need to perform 3 tasks in our Go program.
- One way is sequential execution. We complete Task 1 first then go to Task 2 and then Task 3. Each task waits for previous task to complete.
- This can make our program slow.
- So, we need a better way to handle multiple tasks.

## Concurrency (Structure)

- Managing multiple tasks at once. The tasks don’t have to run at the exact same time, but they are in progress together.
- We keep switching between the tasks so that they are all in porgress.
- A single CPU core can still achieve this by switching between tasks quickly.


## Parallelism (Execution)

- Tasks literally run at the same time.
- Example with multiple CPU cores:

```go
Core 1 → Task A
Core 2 → Task B
Core 3 → Task C
```


## What are Goroutines?

- Goroutines are lightweight threads managed by Go runtime.
- They allow functions to run concurrently.
- Normal threads (like in Java/C++) are heavy. Creating thousands of them is expensive. Goroutines are very lightweight. Go program can easily run millions of goroutines.

## Main may exit early

- If we are calling goroutines from main function, and the main function finishes, the program terminates.
