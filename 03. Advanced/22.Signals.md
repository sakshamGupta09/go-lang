# Signals

- The OS wants to send some messages to our program.
- We will listen for these messages and perform some action.
- Without signals, if someone presses ctrl+c, our process will immediately terminate. With signals, we get a message from the OS so we can run the cleanup and then do shutdown gracefully.

## Common Signals

| Signal    | Meaning                    |
| --------- | -------------------------- |
| `SIGINT`  | Interrupt (usually Ctrl+C) |
| `SIGTERM` | Graceful termination       |
| `SIGKILL` | Force kill immediately     |
| `SIGHUP`  | Reload/restart config      |
| `SIGQUIT` | Quit and dump info         |


## Basic flow

- Create a channel. Whenever OS sends a message, it sends it to this channel.
- Specify which signals we want to listen to. Whenever OS sends me these messages, please send them to the channel
- Wait for OS to send that signal. Program blocks here.
- Do some activity when message comes


## Simple example

```go
package main

import (
	"fmt"
	"os"
	"os/signal"
	"syscall"
)

func main() {

	// Channel to receive signals
	sigChan := make(chan os.Signal, 1)

	// Notify channel when signal arrives
	signal.Notify(sigChan, syscall.SIGINT, syscall.SIGTERM)

	fmt.Println("Program running... Press Ctrl+C")

	// Wait until signal received
	sig := <-sigChan

	fmt.Println("Received signal:", sig)
	fmt.Println("Cleaning up resources...")
	fmt.Println("Shutting down gracefully")
}
```

## Real-World Example: Graceful Server Shutdown

```go
package main

import (
	"fmt"
	"os"
	"os/signal"
	"syscall"
	"time"
)

func main() {

	go func() {
		for {
			fmt.Println("Server handling requests...")
			time.Sleep(2 * time.Second)
		}
	}()

	sigChan := make(chan os.Signal, 1)

	signal.Notify(sigChan, syscall.SIGINT, syscall.SIGTERM)

	// Wait for shutdown signal
	sig := <-sigChan

	fmt.Println("\nReceived:", sig)
	fmt.Println("Closing database...")
	time.Sleep(2 * time.Second)

	fmt.Println("Stopping workers...")
	time.Sleep(1 * time.Second)

	fmt.Println("Server shutdown complete")
}
```
