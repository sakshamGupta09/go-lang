# Deadlock

- It is a situation in which goroutines are not unable to proceed because each goroutine is waiting for other goroutine to release resources.
- This results in a situation where none of the goroutine can make any progress.


```go
func main() {
	var mu1, mu2 sync.Mutex

	go func() {
		mu1.Lock()
		fmt.Println("G1 locked MU1")
		time.Sleep(time.Second)
		mu2.Lock()
		fmt.Println("G1 locked MU2")
		mu1.Unlock()
		mu2.Unlock()
	}()

	go func() {
		mu2.Lock()
		fmt.Println("G2 locked MU2")
		time.Sleep(time.Second)
		mu1.Lock()
		fmt.Println("G2 locked MU1")
		mu2.Unlock()
		mu1.Unlock()
	}()

	select {}

}
```

- G1 runs first and locks MU1. After that it sleeps for 1 second.
- G2 runs and it locks MU2 and goes for sleep.
- G1 wakes up and tries to lock MU2, but MU2 is already locked by G1 so it waits for the lock to be released.
- G2 wakes up and tries to lock MU1 but MU1 is locked by G1 and it keeps waiting for the lock to be released.

## Solution

- Mutexes are safe only if all goroutines follow the same locking order. Always acquire locks in same order.
- Everywhere MU1->mu2
- If order is same, when first G1 runs it locks MU1. Now when G2 runs it cannot acquire any lock because order is same and MU1 is already locked. So it waits without locking any mutex.
