# WaitGroup

- I have N tasks running… let me wait until all of them finish.
- Let's say we have three goroutines. `main` function should not exit early, otherwise goroutines get killed.


## How it works

- A waitGroup has three operations.
- `Add(n)` I am starting n tasks. Always call `Add()` before starting goroutines. Calling it multiple times increments the same shared counter.
- `Done()` One task finished. Please decrease the counter by one.
- `Wait()` Block until counter becomes 0


## Basic Example

```go
func main() {
	var wg sync.WaitGroup

	wg.Add(3)

	go func() {
		time.Sleep(time.Second * 1)
		fmt.Println("First goroutine finished")
		wg.Done()
	}()

	go func() {
		time.Sleep(time.Second * 2)
		fmt.Println("Second goroutine finished")
		wg.Done()
	}()

	go func() {
		time.Sleep(time.Second * 3)
		fmt.Println("Third goroutine finished")
		wg.Done()
	}()

	fmt.Println("Waiting for all goroutines to finish")
	wg.Wait()

	fmt.Println("All goroutiners now finished")
}
```

```go
func main() {
	var wg sync.WaitGroup

	for i := range 3 {
		wg.Add(1)

		go func() {
			defer wg.Done()
			fmt.Println("Starting goroutine number", i)
			time.Sleep(time.Second)
			fmt.Println("Finsished goroutine number", i)
		}()
	}

	fmt.Println("Waiting for all goroutines to finish")
	wg.Wait()

	fmt.Println("All goroutiners now finished")
}
```

## Parallel API calls


```go
func fetchData(id int, wg *sync.WaitGroup) {
	defer wg.Done()
	fmt.Println("Fetching data for ....", id)
	time.Sleep(time.Second * 2)
}

func main() {
	var wg sync.WaitGroup

	for i := range 3 {
		wg.Add(1)

		go fetchData(i, &wg)
	}

	fmt.Println("Waiting for all goroutines to finish")
	wg.Wait()

	fmt.Println("All goroutiners now finished")
}
```


## Worker pool with WaitGroup

- We have multiple tasks but limited fixed number of workers to work on them.
- We want to know when all the tasks will be completed.
- Whenever a new task comes, we can increase the counter and when a goroutine finsishes, we decrease the counter.
