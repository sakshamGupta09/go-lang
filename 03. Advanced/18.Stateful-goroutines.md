# Stateful Goroutines

- We have a shared state and multiple goroutines try to update it. It leads to race conditions and we need to use locks aor atomic counters.
- Instead, one goroutine will own data inside it and logic to update it. Noone from outside can access or modify this data directly.
- Other goroutines can communicate with stateful goroutine using channels. Like they can send a message to read or update the state values.
