# Context

- We have to perform a task. We can simply attach a context to our task which will give us some special abilities like:
- Cancel the ongoing task.
- Timeout or deadline.
- Provide some extra information (like request ID or auth information).


## Create a context

- We can create a root context or parent using two methods `context.TODO` and `context.Background`.
- `context.TODO` is used when we are unsure about which context to use or will replace it with a proper context in future. So, it's just a placeholder for now.
- All other methods like `withTimeout`, `withCancel` and `withDeadline` takes a parent context as parameter.


```go
func main() {
	rootCtx := context.Background()
}
```

## Cancellation

- Whenever we use cancel or timeout, a signal is broadcasted and we can listen to that signal using `<- ctx.Done()`.

```go
func main() {
	rootContext := context.Background()

	ctx, cancel := context.WithCancel(rootContext)

	go func() {
		<-ctx.Done()
		fmt.Println("Stopped ...")
	}()

	cancel()
	time.Sleep(time.Second * 2)
}
```

## Timeout/ Deadline

- Instead of manually calling the `cancel`, start a timer and after some given time call the `cancel` automatically and send a signal.

```go
ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
defer cancel()
```

## Values

```go
ctx := context.WithValue(context.Background(), "userID", 42)

id := ctx.Value("userID")
```
