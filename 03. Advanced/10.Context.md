# Context

- Context is an object that carries information about cancellation, timeouts, deadlines and request scoped values.

## Create a context

- Two methods. `Background` and `TODO`.
- Both behave exactly the same in code. No difference in performance or behavior. The only difference is your intention.
- `context.Background()` → “This is the correct starting point”
- `context.TODO()` → “I don’t know what to use yet, I’ll fix it later”. It's a placeholder.

```go
	ctx1 := context.TODO()
	ctx2 := context.Background()
```


## Pass key value pairs to a context

```go
todoCtx := context.TODO()

ctx := context.WithValue(todoCtx, "name", "John")

fmt.Println(ctx.Value("name"))
```
