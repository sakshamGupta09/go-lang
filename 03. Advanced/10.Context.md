# Context

- Context is used to control lifecycle of a goroutine.
- Context is an object that carries information about cancellation, timeouts, deadlines and request scoped values.
- `context.Context`

## Create a context

- Two methods. `Background` and `TODO`.
- Both behave exactly the same in code. No difference in performance or behavior. The only difference is your intention.
- `context.Background()` → “This is the correct starting point”
- `context.TODO()` → I don’t know which context to use yet, I’ll replace it with another one. It's a placeholder for now.

```go
	ctx1 := context.TODO()
	ctx2 := context.Background()
```


## Pass key value pairs to a context

```go
todoCtx := context.TODO()

ctx := context.WithValue(todoCtx, "name", "John")

fmt.Println(ctx.Value("name"))
```

## context.Done

- Signals cancellation of an operation.
- We create a cotext with a expiry. After that time elapsed, the operation will be maked at cancelled.

```go
func checkEven(ctx context.Context, num int) string {
	select {
	case <-ctx.Done():
		fmt.Println("Operation was cancelled")
		return "Cancelled operation"

	default:
		if num%2 == 0 {
			return "Number is an even number"
		}
		return "number is odd"
	}
}

func main() {
	rootCtx := context.Background()
	ctx, cancel := context.WithTimeout(rootCtx, time.Second)

	time.Sleep(time.Second * 2)
	result := checkEven(ctx, 6)
	defer cancel()

	fmt.Println(result)

}
```
