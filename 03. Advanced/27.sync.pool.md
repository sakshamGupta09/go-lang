# sync.Pool

Instead of creating new objects again and again, goroutines can 

- Take an old object from the pool,
- Use it,
- Clean/reset it,
- and put it back for reuse.

This helps reduce:

- Memory allocations
- Garbage collection pressure
- CPU overhead


## Simple Example

### Without Pool

```go
func process() {
	buf := &bytes.Buffer{}

	buf.WriteString("hello")
	fmt.Println(buf.String())
}
```
- Every call creates a new bytes.Buffer.
- If this runs thousands of times per second → lots of allocations.

### With sync.Pool

```go
var pool = sync.Pool{
	New: func() any {
		fmt.Println("Creating new buffer")
		return new(bytes.Buffer)
	},
}
```

```go
func process() {
	// Get object from pool
	buf := pool.Get().(*bytes.Buffer)

	// IMPORTANT: reset old data
	buf.Reset()

	buf.WriteString("hello")
	fmt.Println(buf.String())

	// Put back into pool
	pool.Put(buf)
}

func main() {
	process()
	process()
	process()
}
```
