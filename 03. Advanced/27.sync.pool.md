# sync.Pool

**Object allocation and garage collection** can be expensive especially if we are dealing with objects of massive size like more than 20 fields. So, Instead of creating new objects again and again, we can -

- Take an old object from the pool,
- Use it,
- Clean/reset it,
- and put it back for reuse.

This helps reduce:

- Memory allocations
- Garbage collection pressure
- CPU overhead


## Simple Example

### Without Pool

```go
func process() {
	buf := &bytes.Buffer{}

	buf.WriteString("hello")
	fmt.Println(buf.String())
}
```
- Every call creates a new bytes.Buffer.
- If this runs thousands of times per second → lots of allocations.

### With sync.Pool

```go
var pool = sync.Pool{
	New: func() any {
		fmt.Println("Creating new buffer")
		return new(bytes.Buffer)
	},
}
```
- If pool is empty, New method is called which creates and returns a fresh object.
- New method is optional. If the pool is empty and there is no new function, we get a nil value from the pool.

```go
func process() {
	// Get object from pool
	buf := pool.Get().(*bytes.Buffer)

	// IMPORTANT: reset old data
	buf.Reset()

	buf.WriteString("hello")
	fmt.Println(buf.String())

	// Put back into pool
	pool.Put(buf)
}

func main() {
	process()
	process()
	process()
}
```

```go
Hello World!
Hello World!
Hello World!
Hello World!
Hello World!
Hello World!
```
- Buffer created only once and reused multiple times.


## How It Works Internally

```go
obj := pool.Get()
```
- checks if reusable object exists in the pool. It works in LIFO manner.
- if yes → returns it
- otherwise → calls New()

```go
pool.Put(obj)
```
- Go stores it for future reuse.


## Struct

```go
type Person struct {
	Name string
	Age  int
}

var pool = sync.Pool{
	New: func() any {
		fmt.Println("Creating a new Person")
		return &Person{}
	},
}
```
