# Channel Synchronization


## Wait for a task to complete

```go
func main() {
	done := make(chan bool)

	go func() {
		fmt.Println("WORK STARTED ....")
		time.Sleep(2 * time.Second)
		done <- true
	}()

	<-done
	fmt.Println("TASK IS NOW COMPLETE")
}
```

```go
func main() {
	goroutinesCount := 3
	done := make(chan int, goroutinesCount)

	for i := range goroutinesCount {
		go func(id int) {
			fmt.Println("STARTING Goroutine number", id)
			time.Sleep(1 * time.Second)
			done <- id
		}(i)
	}

	for range goroutinesCount {
		fmt.Println(<-done)
	}

}
```

## Channels and Range

- `range` keeps reading forever until channel is closed. Sender should always close the channel.
- It is blocking in nature.

```go
func main() {
	done := make(chan int)

	go func() {
		done <- 1
		done <- 2
		done <- 3
	}()

	for value := range done {
		fmt.Println(value)
	}
}
```
- This gives an error because range is continuously creating new receivers as long as the channel is open. After the last sent value, goroutine finishes but receiver is still waiting.
