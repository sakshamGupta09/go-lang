# Unbuffered channel

- It is a channel with no storage. It cannot hold values.
- It only passes data directly from sender → receiver. Sender and receiver must be ready at the same time. Value is instantly passed to receiver.
- I will send only when you are ready to receive.


## Example

```go
package main

import "fmt"

func main() {
	ch := make(chan int)

	go func() {
		ch <- 10 // sending
		fmt.Println("Sent")
	}()

	value := <-ch // receiving
	fmt.Println("Received:", value)
}
```

- Goroutine tries to send 10
- It blocks (waits) until someone receives
- Main goroutine receives <-ch
- Value is transferred
- Both continue


## Blocking behavior

- Since, there is no mechanism to store values, sender and receiver must be ready at the same time.
- If sender tries to send the value, but no receiver is ready, program will crash.
- Similarly, if receiver is ready to receive the data, but there is no sender, program will crash.

### No receiver

```go
ch := make(chan int)
ch <- 10 // DEADLOCK
```

### No sender

```go
ch := make(chan int)
value := <-ch // DEADLOCK
```

## Wait for a task to finish:

```go
package main

import (
	"fmt"
	"time"
)

func worker(done chan bool) {
	fmt.Println("Working...")
	time.Sleep(2 * time.Second)

	done <- true // signal completion
}

func main() {
	done := make(chan bool)

	go worker(done)

	<-done // wait here until worker finishes
	fmt.Println("Work completed")
}
```
