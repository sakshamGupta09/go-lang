# Unbuffered channel

- It is a channel with no storage. It cannot hold data.
- It only transfers data directly from sender to receiver.
- So, communication happens only when both are ready at the same time.
- If any of them is not ready, we wait and block execution till the other party is ready.

## Create a unbuffered channel

```go
ch := make(chan int) // unbuffered channel
```

## Basic Example

```go
func main() {
    ch := make(chan int)

    go func() {
        ch <- 10 // send
        fmt.Println("Sent")
    }()

    value := <-ch // receive
    fmt.Println("Received:", value)
}
```
- There are two goroutines. Main and function.
- Main goroutine waits for a sender to send data and execution is blocked.
- Function goroutine sends the data and finishes.
- Data is immediately transferred to receiver and it's execution continues.


## Blocking behavior

```go
func main() {
	ch := make(chan string)

	ch <- "Hello World"

	value := <-ch
	fmt.Println(value)
}
```
- We are trying to send a value to an unbuffered channel.
- But the execution is blocked as there is no goroutine to receive that value.
- It will never go to the next line and throw an error.

### No receiver

```go
func main() {
	ch := make(chan int)
	ch <- 10
}
```

### No sender

```go
func main() {
	ch := make(chan int)
	value := <-ch

	fmt.Println(value)
}
```

## Wait for task to be completed

```go
func doWork(ch chan bool) {
	time.Sleep(time.Second * 2) // Long running task
	ch <- true
}

func main() {
	ch := make(chan bool)

	go doWork(ch)

	<-ch
	fmt.Println("Work completed")
}
```
