# Mutexes

## Shared Data + Multiple Goroutines

- We have some shared data and multiple goroutines are updating that data in same time.
- This can lead to race conditions.
- Ideally when one goroutine updates sahared data, next goroutine should work on the updated value.

```go
func main() {
	var wg sync.WaitGroup

	counter := 0

	for range 1000 {
		wg.Add(1)
		go func() {
			defer wg.Done()
			counter++
		}()
	}

	wg.Wait()
	fmt.Println(counter)
}
```
- Here output will keep varying.

## Mutex (Mutual Exclusion)

- Mutex ensures that only ONE goroutine can access this critical section at a time.
- Think of it like a bathroom lock, If someone is inside → others must wait. When they leave → next person enters.
