# Multiplexing

- We want to listen to multiple channels but react to the first sender only.

## Select

```go
select {
case msg := <-ch1:
    fmt.Println("Received from ch1:", msg)

case msg := <-ch2:
    fmt.Println("Received from ch2:", msg)
}
```

- It blocks until one case is ready.
- Executes only one case.
- If multiple are ready → randomly picks one.


## Fastest response wins

```go
func main() {
	ch1 := make(chan string)
	ch2 := make(chan string)

	go func() {
		time.Sleep(2 * time.Second)
		ch1 <- "from ch1"
	}()

	go func() {
		time.Sleep(1 * time.Second)
		ch2 <- "from ch2"
	}()

	select {
	case msg := <-ch1:
		fmt.Println(msg)
	case msg := <-ch2:
		fmt.Println(msg)
	}
}
```

## Non-blocking channel

- Time at which select statement ran, if no channel emitted, we will go to default case.
- It's non blocking because it doesn't wait for any channel to send a value.


```go
select {
case msg := <-ch:
	fmt.Println("Received:", msg)
default:
	fmt.Println("No message available")
}
```

## Timeout handling

- If no response within 2 seconds, timeout

```go
select {
case msg := <-ch:
	fmt.Println("Received:", msg)

case <-time.After(2 * time.Second):
	fmt.Println("Timeout!")
}
```

## How to receive multiple values

- Select waits for multiple channels and executes the first message case.
- But what if we want to receive multiple channel values.
- We can use a for loop.

```go
func main() {
	ch1 := make(chan int)
	ch2 := make(chan int)
	ch3 := make(chan int)

	go func() {
		time.Sleep(3 * time.Second)
		ch1 <- 1
	}()

	go func() {
		time.Sleep(2 * time.Second)
		ch2 <- 2
	}()

	go func() {
		time.Sleep(1 * time.Second)
		ch3 <- 3
	}()

	for range 3 {
		select {
		case msg := <-ch1:
			fmt.Println("CHANNEL 1", msg)
		case msg := <-ch2:
			fmt.Println("CHANNEL 2", msg)
		case msg := <-ch3:
			fmt.Println("CHANNEL 3", msg)
		}
	}

	fmt.Println("CODE COMPLETED IN MAIN")
}
```

- When a channel is closed, select will receive the zero value from the channel on which we can do the cleanup activities.

```go
func main() {
	ch1 := make(chan int)

	go func() {
		time.Sleep(2 * time.Second)
		ch1 <- 1
		close(ch1)
	}()

	for {
		select {
		case msg, ok := <-ch1:
			if !ok {
				fmt.Println("Channel closed")
				return
			}
			fmt.Println("CHANNEL 1", msg)
		}
	}

	fmt.Println("CODE COMPLETED IN MAIN")
}
```

## Sending using select

- Multiple channels are trying to send value.
- Whichever receiver gets ready first will get the chance to execute.


```go
select {
case ch1 <- "hello":
	fmt.Println("Sent to ch1")

case ch2 <- "world":
	fmt.Println("Sent to ch2")
}
```

- A send case runs only if There is a receiver ready on that channel.


```go
func main() {
	ch1 := make(chan string)
	ch2 := make(chan string)

	go func() {
		fmt.Println(<-ch2) // only ch2 is receiving
	}()

	select {
	case ch1 <- "hello":
		fmt.Println("Sent to ch1")

	case ch2 <- "world":
		fmt.Println("Sent to ch2")
	}
}
```


## Non-blocking send

```go
select {
case ch <- 10:
	fmt.Println("Sent value")

default:
	fmt.Println("Channel not ready, skipping")
}
```
- If no receiver is ready, It won’t block Goes to default.

## Timeout while sending

```go
select {
case ch <- 10:
	fmt.Println("Sent")

case <-time.After(2 * time.Second):
	fmt.Println("Send timeout")
}
```
- If channel is not ready within 2 sec:Timeout triggers


## Worker pool (real-world usage)

```go
jobs := make(chan int)
done := make(chan bool)

go func() {
	for {
		select {
		case jobs <- 1:
			fmt.Println("Job sent")

		case <-done:
			fmt.Println("Stopping sender")
			return
		}
	}
}()

time.Sleep(2 * time.Second)
done <- true
```
