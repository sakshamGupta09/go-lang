# sync.NewCond

- It's a notification system. Goroutines go to sleep and wait for a signal to arrive. After signal comes, they wake up and resume execution.
- A `sync.Cond` has 3 important methods and it always works with a mutex.

| Method        | Meaning                     |
| ------------- | --------------------------- |
| `Wait()`      | Sleep until notified        |
| `Signal()`    | Wake one waiting goroutine  |
| `Broadcast()` | Wake all waiting goroutines |

## Mental Model

```go
Mutex = protects shared state

Cond = notification system saying:
       "Hey, state changed.
        Wake up and check again."
```
- sync.Cond does NOT tell you what changed or whether condition is true.
- It only says:“Something may have changed.”

## Wait

Wait methods does three things -

- Unlocks the mutex. I am going to sleep now, but I will release teh lock before going. Other goroutines can now modify data.
- Puts goroutine to sleep. No CPU wasted.
- When signaled, wakes up, re-locks mutex automatically and continues execution.


## Why use for loop around Wait?

Always do

```go
for !condition {
    cond.Wait()
}
```
NOT

```go
if !condition {
    cond.Wait()
}
```

- Because waking up does NOT guarantee condition became true.
- So after waking up, check again and if condition still false → sleep again

## Basic Example

```go
package main

import (
	"fmt"
	"sync"
	"time"
)

func main() {
	var mu sync.Mutex
	cond := sync.NewCond(&mu)

	ready := false

	// Waiting goroutine
	go func() {
		mu.Lock()

		for !ready {
			fmt.Println("Waiting...")
			cond.Wait()
		}

		fmt.Println("Received signal!")
		mu.Unlock()
	}()

	time.Sleep(2 * time.Second)

	// Producer goroutine
	mu.Lock()
	ready = true
	fmt.Println("Sending signal...")
	cond.Signal()
	mu.Unlock()

	time.Sleep(time.Second)
}
```

## Step-by-step intuition

```go
Goroutine:
    lock mutex
    check condition

    condition false?
        yes -> sleep using Wait()

Producer:
    lock mutex
    change shared state
    Signal()

Waiting goroutine:
    wakes up
    re-locks mutex
    checks condition again
```


## Producer Consumer

```go
package main

import (
	"fmt"
	"sync"
	"time"
)

func main() {
	var mu sync.Mutex
	cond := sync.NewCond(&mu)

	queue := []int{}

	// Consumer
	go func() {
		mu.Lock()

		for len(queue) == 0 {
			fmt.Println("Queue empty. Waiting...")
			cond.Wait()
		}

		item := queue[0]
		queue = queue[1:]

		fmt.Println("Consumed:", item)
		mu.Unlock()
	}()

	time.Sleep(2 * time.Second)

	// Producer
	mu.Lock()
	queue = append(queue, 42)
	fmt.Println("Produced: 42")

	cond.Signal()
	mu.Unlock()

	time.Sleep(time.Second)
}
```

## Combining Data, cond and mutex inside a single struct

- Instead of scattered methods and variables, everything is encapsulated in one place.

```go
type Buffer struct {
	queue []int
	mu    sync.Mutex
	cond  *sync.Cond
}

func NewBuffer(size int) *Buffer {
	b := &Buffer{
		queue: make([]int, 0, size),
	}
	b.cond = sync.NewCond(&b.mu)
	return b
}

func (b *Buffer) Produce(item int) {
	b.mu.Lock()
	for len(b.queue) == MAX_BUFFER_SIZE {
		b.cond.Wait()
	}
	b.queue = append(b.queue, item)
	b.cond.Signal()
	b.mu.Unlock()
}

func (b *Buffer) Consume() {
	b.mu.Lock()
	defer b.mu.Unlock()

	for len(b.queue) == 0 {
		fmt.Println("QUEUE IS empty, Please wait")
		b.cond.Wait()
	}
	item := b.queue[0]
	fmt.Println("CONSUMED A VALUE", item)
	b.queue = b.queue[1:]
	b.cond.Signal()
}

func main() {
	b := NewBuffer(MAX_BUFFER_SIZE)
	var wg sync.WaitGroup

	wg.Add(2)

	go Producer(b, &wg)
	go Consumer(b, &wg)

	wg.Wait()}

```

## Signal vs Broadcast

- Signal wakes up one goroutine and asks it to recheck the condition.
- Broadcast Wakes up ALL waiting goroutines.
