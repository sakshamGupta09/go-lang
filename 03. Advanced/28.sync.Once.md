# sync.Once

- Multiple goroutines have a common piece of code.
- We want that piece of code to run only one time despite multiple goroutines trying to run it.
- While the code is in progress, other goroutines will wait.
- sync.Once lets many goroutines race to perform an action, but guarantees that exactly one of them actually executes it.


## Basic Example

```go
package main

import (
	"fmt"
	"sync"
)

var once sync.Once

func initialize() {
	fmt.Println("Initializing...")
}

func main() {
	for i := 0; i < 5; i++ {
		go func() {
			once.Do(initialize)
		}()
	}

	var input string
	fmt.Scanln(&input)
}
```
- Imagine 5 goroutines arrive simultaneously:

```go
G1 -> once.Do(init)
G2 -> once.Do(init)
G3 -> once.Do(init)
G4 -> once.Do(init)
G5 -> once.Do(init)
```

```go
G1 sees init not done
G1 runs init()

G2 waits
G3 waits
G4 waits
G5 waits
```
After init() completes:

```go
done = true
```
Now every future call simply returns immediately.

```go
G2 -> return
G3 -> return
G4 -> return
G5 -> return
```

## Common Use Case: Singleton Initialization

```go
func GetDB() *DB {
	return connectToDB()
}
```
- Suppose creating a database connection is expensive.
- Without sync.Once, If 100 goroutines call this simultaneously, 100 database connections created which is not good.

```go
var (
	db   *DB
	once sync.Once
)

func GetDB() *DB {
	once.Do(func() {
		db = connectToDB()
	})

	return db
}
```
- First call  -> create DB connection. All others  -> reuse same connection

```go
once.Do(func() {
	fmt.Println("Hello")
})

once.Do(func() {
	fmt.Println("World")
})
```
- Because the first successful Do() call permanently marks the Once as done.
